## STEP 1 — Add QStash env vars to Vercel (CRITICAL — enables sub-minute callbacks)

**What:** Three env vars that connect the deployed app to Upstash QStash.
**Who:** Anyone with access to the Vercel project and Upstash account.
**Time:** ~3 minutes.

### 1a. Get the keys from Upstash
1. Go to https://console.upstash.com/qstash and sign in.
2. On the QStash page, find the **Keys** section. Copy:
   - `QSTASH_TOKEN` — the publish token (starts with `eyJ...` or similar)
   - `QSTASH_CURRENT_SIGNING_KEY` — labeled "Current Signing Key"
   - `QSTASH_NEXT_SIGNING_KEY` — labeled "Next Signing Key"

### 1b. Add them to Vercel
1. Go to https://vercel.com → open the Earlymark/Assistantbot project.
2. Click **Settings** → **Environment Variables**.
3. Add these three variables for **Production** (and Preview if desired):
   ```
   QSTASH_TOKEN              = <value from 1a>
   QSTASH_CURRENT_SIGNING_KEY = <value from 1a>
   QSTASH_NEXT_SIGNING_KEY    = <value from 1a>
   ```
4. **Confirm** `NEXT_PUBLIC_APP_URL` is already set (it should be). QStash
   will call back to `${NEXT_PUBLIC_APP_URL}/api/qstash/callback`.
5. Click **Redeploy** (or push any commit) to pick up the new vars.

### Verify it worked
After redeploy, hit `GET /api/check-env` or look at the admin integration-readiness
view — **qstash** should report `ready: true`.

## STEP 2 — Add QStash secrets to GitHub Actions (OPTIONAL — for CI/future use)

**What:** Same three keys as Step 1, but added as GitHub repo secrets so CI
jobs and future workflows can reference them.
**Who:** Anyone with repo admin access on GitHub.
**Time:** ~2 minutes.

1. Go to https://github.com/michaelwu1407-droid/assistantbot → **Settings** →
   **Secrets and variables** → **Actions**.
2. Click **New repository secret** for each:
   ```
   QSTASH_TOKEN
   QSTASH_CURRENT_SIGNING_KEY
   QSTASH_NEXT_SIGNING_KEY
   ```
   (Same values as Step 1.)

**Note:** The app itself reads these from Vercel — not from GitHub. This step is
only needed if you want workflows to reference QStash directly. It is NOT required
for delayed callbacks to work.

## STEP 3 — Set beta password for the Map feature

**What:** One env var that controls who can access the Map page.
**Who:** Anyone with Vercel access.

In Vercel → Settings → Environment Variables, add:
```
NEXT_PUBLIC_MAP_BETA_PASSWORD = <choose a short password to share with beta testers>
```

If this is left unset, the map gate accepts any input (wide open). Set it to
restrict access to invited testers only.
